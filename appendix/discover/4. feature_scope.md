# GLP-1 치료 관리 MVP 요구사항 문서 (Phase 0)

## 1. 제품 개요 및 비전

### 1.1. 제품 개요
본 제품은 GLP-1 주사제 사용자들을 위한 디지털 솔루션으로, 복잡한 **개인별 투여 스케줄**을 자동 관리하고, **주요 증상 및 체중 변화 기록**을 데이터 기반의 보고서로 자동 생성하여 의료진과의 커뮤니케이션 효율과 환자의 치료 안전성을 극대화하는 것을 목표로 한다.

### 1.2. 비전
GLP-1 치료 과정에서 발생하는 모든 불확실성(스케줄 혼동, 증상 보고의 비일관성)을 해소하여, 사용자가 안전하고 효율적으로 치료 목표에 도달할 수 있도록 돕는 필수적인 디지털 파트너가 된다.

---

## 2. 타겟 사용자와 핵심 시나리오

### 2.1. 타겟 사용자
GLP-1 주사제를 복용하는 사용자 (자가 투여에 익숙하지만, 스케줄 관리와 데이터 기록에 비효율을 겪는 모든 연령대의 사용자).

### 2.2. 핵심 시나리오 (P0 기능 기반)

| 시나리오 No. | 시나리오 | 해결되는 Pain Point/가설 |
| :--- | :--- | :--- |
| **SC1. 유연한 스케줄 변경** | 사용자가 심한 부작용으로 인해 의사의 진료를 받았고, 의사가 다음 투여 시점을 1주일 연기하도록 지시했다. 사용자는 앱에 변경된 투여일과 사유를 입력하고, **이후의 모든 증량 스케줄이 자동으로 재조정**된다. | P2 (투여 일정 혼동), H2 (자동 스케줄러) |
| **SC2. 의료진 보고서 제출** | 사용자가 정기 검진을 위해 병원을 방문한다. 방문 직전에 앱에서 지난 4주간의 **투여 용량 변화, 체중 변화 추이, 부작용 강도**가 시각화된 **PDF 보고서**를 즉시 생성하여 의료진에게 제출한다. | P5 (의료진 소통 비효율성), H5 (간편 기록/보고서) |

## 3. 기능 목록

| ID | 기능명 | 설명 | 우선순위 |
| :--- | :--- | :--- | :--- |
| **F001** | **개인화된 투여 스케줄러** | 초기 처방 정보(시작일, 증량 테이블)를 기반으로 전체 스케줄을 자동 생성한다. 사용자/의료진 입력에 따라 다음 투여일 또는 용량을 수동으로 변경할 경우, 이후의 모든 스케줄과 알림을 즉시(1초 이내) 자동 재계산하여 반영해야 한다. | **P0** (MVP Core) |
| **F002** | **주요 증상 및 체중 간편 기록** | 투여일 및 매일의 **체중**과 **주요 부작용 (메스꺼움, 구토, 변비 등)**의 심각도를 **5점 척도(VAS)**로 최소한의 터치로 간편하게 기록할 수 있도록 한다. 식사 및 식단 기록은 포함하지 않는다. | **P0** (MVP Core) |
| **F003** | **자동 의료진 보고서 생성** | F001의 용량 변화 시점, F002의 체중 변화 추이 그래프, 부작용 강도/빈도 시각화 그래프를 통합한 **PDF 형식의 보고서**를 3초 이내에 자동 생성하고 내보낼 수 있어야 한다. | **P0** (MVP Core) |
| **F004** | **공식 부작용 대처 가이드** | 의료기관에서 검증된 정보를 기반으로 주요 부작용별 **공식적 대처 가이드라인**을 제공하여 사용자의 치료 중단 고려율을 낮추고 안전성을 확보한다. | **P0** (안전성 확보) |
| **F005** | **단백질 섭취 목표 설정** | 사용자(체중/활동량) 정보를 기반으로 일일 단백질 섭취량 목표를 설정하고, 목표 달성 여부를 표시할 수 있어야 한다. (수동 입력 또는 간단한 섭취량 체크 기반) | **P1** (High Utility) |

## 4. 비기능 요구사항

| 분류 | ID | 요구사항 | 목표 수치/지표 |
| :--- | :--- | :--- | :--- |
| **성능** | NFR01 | 스케줄러 자동 재계산 시간 | 1초 이내 |
| | NFR02 | 의료진 보고서(PDF) 생성 시간 | 3초 이내 |
| **보안** | NFR03 | 민감 정보 보호 | 개인 건강 정보(PHI)에 대한 국내외 보안 표준 준수 |

| **유용성** | NFR05 | 사용 편의성 | 핵심 기록 기능(F002)은 3회 이내의 터치로 완료 가능해야 함. |
| **확장성** | NFR06 | 데이터 모델 | 향후 다른 GLP-1 약물(투여 주기, 용량 테이블 상이) 및 타 기능(H3, H4 확장)을 수용할 수 있는 구조적 확장성을 확보해야 함. |

## 5. 제약사항 및 성공 지표

### 5.1. 제약사항
1.  **범위 통제:** 상세한 식단 기록 (Full Meal Logging) 및 일반 사용자 간의 공개 커뮤니티(Open Forum) 기능은 MVP 범위에서 **제외**한다.
2.  **정보 출처:** 부작용 대처 가이드(F004)에 제공되는 모든 정보는 **의료 전문가 또는 공식 기관에서 검증된 내용**만을 사용해야 한다.

### 5.2. 성공 지표 (Go Metrics)
| 지표 | 목표 |
| :--- | :--- |
| **H2:** 주사 투여일 오차율 감소 | **15% 감소** |
| **H5:** 의사와의 상담 만족도 개선 | **1점 개선** (Likert 5점 척도 기준) |
| **안전성:** 부작용으로 인한 치료 중단 고려율 | 20% 미만 유지 (추적 지표) |
