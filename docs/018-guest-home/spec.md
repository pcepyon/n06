# 비로그인 홈 화면 (Guest Home) 스펙

## 개요

앱 스토어 심사 통과 및 신규 사용자 전환을 위한 비로그인 상태 홈 화면.
GLP-1 치료의 과학적 근거와 앱의 가치를 전달하여 회원가입을 유도한다.

**목표**:
- 앱 스토어 심사 요건 충족 (로그인 없이 콘텐츠 제공)
- GLP-1 치료에 대한 신뢰 구축
- 앱의 핵심 가치 전달로 회원가입 전환율 향상
- PRD 감정적 UX 원칙에 따른 "두려움 → 희망" 전환

---

## 화면 구조

```
[비로그인 홈 - SingleChildScrollView]
│
├── 1. 환영 섹션 (WelcomeSection)
│   └── 시간대별 인사 + 희망 메시지
│
├── 2. GLP-1 과학적 근거 섹션 (ScientificEvidenceSection)
│   ├── 섹션 헤더
│   ├── 효과 카드 캐러셀 (5개)
│   └── Science 선정 메시지
│
├── 3. 치료 여정 미리보기 섹션 (JourneyPreviewSection)
│   └── 12주 타임라인 시각화
│
├── 4. 앱 기능 소개 섹션 (AppFeaturesSection)
│   └── 핵심 기능 5개 카드
│
├── 5. 부작용 대처 가이드 섹션 (SideEffectsGuideSection)
│   └── 주요 증상 4개 미리보기 카드

    ├── Primary: 회원가입 버튼
    └── Secondary: 더 둘러보기 링크
```

## 섹션 3: 치료 여정 미리보기 (JourneyPreviewSection)

### 목적

- 두려움 해소
- 현실적 기대치 설정

### UI 구성

#### 섹션 헤더
```
나의 12주 치료 여정,
미리 살펴볼까요?

GLP-1 치료는 보통 낮은 용량에서 시작해
몸이 적응하면 점차 용량을 높여갑니다.
각 단계에서 어떤 변화가 일어나는지 알면

#### 타임라인 항목

##### 1~2주차: 적응의 시작
```
📍 1~2주차: 적응의 시작
─────────────────────────────────────────────

몸이 새로운 신호에 적응하는 시간이에요.

💭 이런 느낌이 들 수 있어요
   • 식사량이 자연스럽게 줄어드는 느낌
   • 가벼운 메스꺼움이나 더부룩함
   • 평소보다 빨리 포만감을 느낌

💚 가비움의 한마디
   "이 증상들은 약이 제대로 작용하고 있다는
    신호예요. 보통 1~2주 내에 나아져요."
```

##### 3~4주차: 첫 번째 변화
```
📍 3~4주차: 첫 번째 변화
─────────────────────────────────────────────

많은 분들이 첫 번째 의미 있는 변화를 느끼는 시점.

💭 기대할 수 있는 변화
   • 체중 감소 시작 (평균 3~5% 전후)
   • 식욕 조절이 한결 수월해짐
   • 초기 적응 증상이 줄어듦
   • 혈당 수치 개선 (해당 시)

💚 가비움의 한마디
   "작은 변화도 큰 의미가 있어요.

#### 2-3. Auto-Slide with Touch Pause [P0]

```dart
// 자동 슬라이드
- 간격: 5초
- 사용자 터치 시: 타이머 정지
- 터치 종료 3초 후: 타이머 재개

Timer? _autoSlideTimer;

void _startAutoSlide() {
  _autoSlideTimer = Timer.periodic(Duration(seconds: 5), (_) {
    _pageController.nextPage(
      duration: Duration(milliseconds: 500),
      curve: Curves.easeInOut,
    );
  });
}

void _onUserInteraction() {
  _autoSlideTimer?.cancel();
  // 3초 후 재개
  Future.delayed(Duration(seconds: 3), _startAutoSlide);
}
```

### 성능 고려사항

1. **애니메이션 최적화**
   - `const` 위젯 최대 활용
   - `RepaintBoundary`로 애니메이션 영역 분리
   - 불필요한 리빌드 방지 (`Selector`, `Consumer`)

2. **Visibility Detection**
   - `visibility_detector` 패키지 또는 `ScrollController` 활용
   - 한 번만 트리거되는 애니메이션은 상태 저장

3. **Image 최적화**
   - 미리보기 이미지는 적절한 해상도로 제공
   - `precacheImage`로 사전 로딩
